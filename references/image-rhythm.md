# 图片使用规范与主题节奏规划

## 图片使用规范

### 图片来源优先级

1. **用户提供的图片**（最优先）— 用用户自己的素材最真实
2. **AI 生成的示意图**（次选）— 用 SVG/CSS 画的图，清晰且不需要外部加载
3. **Unsplash 免费图库**（备选）— 通过 `https://source.unsplash.com/` 获取，只用免费可商用的
4. **纯 CSS 图形**（兜底）— 用 CSS 画圆、矩形、图标，不依赖任何外部资源

### 图片比例标准

| 用途 | 推荐比例 | 建议尺寸 |
|---|---|---|
| 全屏背景（hero 风格） | 16:9 | 1920×1080 |
| 配图（卡片内） | 4:3 或 3:2 | 800×600 |
| 头像/小图标 | 1:1 | 200×200 |

### 图片命名规则

```
{页码}_{用途}.{格式}
例：03_案例图.jpg, 07_流程图.svg
```

### 注意事项

- 所有外部图片使用 `loading="lazy"` 延迟加载
- 全屏背景图必须加深色遮罩（`background: rgba(0,0,0,0.5)`）保证文字可读
- 不要用网络图片做关键内容（万一没网就看不到了），关键图表用 SVG/CSS 画

## 主题节奏规划

如果一份 PPT 所有页都是白底黑字，观众 3 分钟就腻了。用三种页面类型交替制造节奏感：

### 三种页面类型

| 类型 | 背景 | 用途 | 占比建议 |
|---|---|---|---|
| **Light（浅色页）** | 白色/浅灰/米白 | 铺干货、展示数据、讲案例 | ~50% |
| **Dark（深色页）** | 深蓝/纯黑/深灰 | 章节幕封、金句、数据大字报 | ~25% |
| **Hero（全屏图）** | 图片+深色遮罩 | 封面、结尾、情绪转折点 | ~25% |

### 节奏规则（硬规则）

1. **不要连续 3 页同类型** — 最多 2 页 Light 后必须换 Dark 或 Hero
2. **章节幕封必用 Dark** — 每个新章节开头用深色幕封页，给观众"换气"的信号
3. **金句页用 Dark 或 Hero** — 金句不要在普通白底上，没有冲击力
4. **封面和结尾必用 Hero** — 第一印象和最后印象最重要

### AI 生成时如何执行

在 Phase 3.1 选定骨架后，按顺序过一遍：
```
第1页 → 封面 → Hero
第2页 → 目录 → Light
第3页 → 数据大字报 → Dark（连续Light=0，可以不换）
第4页 → 章幕封 → Dark（⚠️连续Dark=2，下一页必须Light）
第5页 → 内容页 → Light
...
```

发现连续 2 页同类型就在下一行标注 `⚠️`，然后强制切换。
